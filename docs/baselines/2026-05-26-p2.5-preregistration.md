# P2.5 Pakistani Dose-Response — PREREGISTRATION

**Branch:** `feat/p2.5-pakistani-dose` (off `main` tip `8159106`)
**Date written:** 2026-05-26 (committed BEFORE any model is trained — the P2.0 lesson: fix the
gate and the decision rule before you can see the result).

## The question (and the lead caveat)

P0→P2 converged on a finding: image-only model surgery on a FROZEN Rad-DINO backbone (P1 head swap,
P2 backbone-family spike, P2.0b max-readout battery) does NOT close the Pakistani external gap. Every
frozen-Rad-DINO readout circles a ~0.78–0.81 external ceiling while hitting 0.90+ cross-source. The
working hypothesis is therefore **domain shift**, not a backbone or pooling deficiency.

This experiment tests the pivotal alternative: **does adding IN-DOMAIN (Pakistani) data to TRAINING
close the gap, and HOW MUCH is needed?** The user explicitly approved spending part of the Pakistani
holdout for this ("dropping a folder acceptable").

**LEAD CAVEAT (carry into every writeup):** Once Pakistani enters training, the held-out Pakistani
test set is a **WITHIN-COHORT (same-site) holdout, NOT a fresh external site.** This experiment measures
**DATA-CLOSABILITY + dose-response**, NOT a new external-generalization claim. A rise in "external
AUROC" here means **"in-domain data closes the same-site gap,"** NOT "we now generalize to new sites."
No WHO-90/70 language, no new-external-site claim.

## The split (FIXED before any run)

- Pakistani (`data/features_mendeley_pk.npz`) is **one-image-per-patient** — confirmed: 3,008 images =
  3,008 unique `patient_id`, max 1 image/patient. So a **patient-disjoint split == image-disjoint
  split.** Zero patient/image leakage by construction.
- Label distribution: **2,494 TB+ (label 1) / 514 normal (label 0)** — heavily TB+-skewed.
- `has_box` is all False, `grid_label` all zeros (no box supervision — same as NIH normals).
- **FIXED TEST SET = 40% of Pakistani patients, stratified by label, seed=7.** Expected sizes:
  ~998 patients (~998 TB+ ≈ 0.40×2494, ~206 normal ≈ 0.40×514). This test set is **NEVER trained on at
  any dose.** Held fixed across all 5 doses.
- **TRAINING POOL = the other 60%** (~1,810 patients: ~1,496 TB+ / ~308 normal).

## The doses (cumulative, nested, seed=7)

Drawn from the 60% training pool, stratified to match the pool's TB+/normal ratio, **nested** so larger
doses contain smaller ones (seed=7 deterministic):

| Dose | Pakistani train rows added | TB+/normal (approx, pool ratio ~83%/17%) |
|---|---|---|
| 0 (baseline) | 0 | — (4 source datasets only) |
| 150 | 150 | ~124 / ~26 |
| 300 | 300 | ~249 / ~51 |
| 600 | 600 | ~497 / ~103 |
| ALL | ~1,810 (full 60% pool) | ~1,496 / ~308 |

Training set at each dose = the 4 source datasets (`data/features.npz`, 13,092 rows: montgomery /
qatar / shenzhen / tbx11k) + d Pakistani training-pool rows. Pakistani rows: `source =
'mendeley_pk_train'`, fresh `group` ids past the existing max (so the dup-cluster guard and the
single-class LODO-skip handle them; `mendeley_pk_train` is mixed-label so it is not a degenerate fold).
`grid_label` = zeros, `has_box` = False (synthesized exactly as `load_nih14_normals()` does).

## The model (held fixed across doses)

Train ONLY the deployed head configuration: **zonal-soft-OR** (`head_kind='zonal-softor'`, the default),
levers `{fusion, zonal, box}`, IDENTICAL recipe + PRIOR-FIXED knobs (g_min=0.25, zone_prior_init=0.4,
r0=5.0/cap 8.0, lambda_box=0.2, lambda_distill=0.2, lambda_tv=0.01, distill_T=2.0). Seed fixed. **SKIP**
the full T2 ablation sweep — train just the one deployed head per dose to keep each dose cheap.

## Calibration (PER-CONFIG — NOT the frozen locked protocol)

This is the one deliberate departure from P1's frozen-locked protocol, and it is methodologically
required here: in P1 Pakistani was an EVAL surface only, so the deployed operating point (frozen
T=1.5915 / thr=0.6105) was the right reference. In P2.5 Pakistani is IN training at varying doses, so
the deployed operating point is no longer the model's own. **Fit T + thr@95sens ONCE per dose on the
seed=7 cal slice of THAT model's own 4-source LODO OOF** (`make_calibration_split`, seed=7, cal_frac=0.20).
Per-config, never reused from another model (the P1 threshold-transfer trap), and the cal slice is the
**4-source OOF only** — the Pakistani test set is NEVER used to fit T or thr.

## The metrics

**On the FIXED Pakistani test set** (patient-level == image-level here):
- patient-level bootstrap AUROC (95% CI, 2000 resamples)
- specificity at sens=0.90 (on the Pakistani test set's own ROC)
- sensitivity at spec=0.70 (on the Pakistani test set's own ROC)
- ECE (10-bin)

**On the 4-source LODO eval slice** (the seed=7 80% eval complement — the FORGETTING check):
- AUROC
- sensitivity at the per-config fitted thr@95sens

## The decision rule (FIXED before the result)

Report the dose-response curve. Identify the dose (if any) at which Pakistani test AUROC crosses **~0.85**
(meaningfully above the ~0.78 ceiling, toward the ~0.90 cross-source level), and whether it **plateaus**.

Pre-stated readings:
- **A monotone rise toward ~0.90 with dose** ⇒ the gap is **data-closable** (in-domain data is the lever).
  Report roughly how many in-domain images it takes to cross ~0.85.
- **A flat curve even at full dose** ⇒ something deeper than data volume (label noise / irreducible
  radiographic overlap / a domain shift no amount of this cohort fixes).

## Drift tripwires (EXPERIMENT_LOG §B — STOP and record if any fires)

- **§B.2 worst/eval-fold sensitivity drop (FORGETTING):** if adding in-domain Pakistani data craters the
  4-source LODO sensitivity (below the ~0.846 baseline floor), STOP and record — in-domain data must
  not destroy source performance.
- **§B.5 a knob tuned on the test set:** T/thr fitted ONLY on the 4-source OOF cal slice, never on the
  Pakistani test set. The Pakistani test set is touched only at final scoring.
- **§B.1 AUROC jump > +0.05:** expected here BY DESIGN (that is the experiment's hypothesis), so a rise
  is not a leakage flag IF the split is clean (patient-disjoint, test set never trained). The honesty
  check is the lead caveat: a rise = same-site closability, NOT new-site generalization.
